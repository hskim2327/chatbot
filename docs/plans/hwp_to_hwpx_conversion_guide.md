# HWP → HWPX 변환 자동화 가이드

## 목적

RFP 원본 `.hwp` 파일을 실제 `.hwpx` 포맷으로 변환한다.  
이 문서는 **HWP를 HWPX로 변환하는 과정까지만** 다룬다.

주의: 확장자만 `.hwp`에서 `.hwpx`로 바꾸는 것은 변환이 아니다. 반드시 한컴오피스에서 문서를 열고 **다른 이름으로 저장 → HWPX 형식**으로 저장해야 한다.

## 작업 환경

- Windows PC
- 한컴오피스 설치 필요
- Python 또는 PowerShell 자동화 사용 가능
- 원본 HWP 파일과 변환된 HWPX 파일은 폴더를 분리한다.

권장 폴더 구조:

```text
project_2nd/
├─ data/
│  ├─ original_hwp/   # 원본 .hwp
│  └─ hwpx_690/       # 변환된 .hwpx
└─ outputs/
   └─ hwpx_conversion/
      ├─ conversion_log.csv
      └─ conversion_failed.csv
```

## 1단계. 원본 파일 준비

- 변환 대상 `.hwp` 파일을 한 폴더에 모은다.
- 파일명은 가능하면 변경하지 않는다.
- 이미 변환된 `.hwpx` 파일과 섞지 않는다.

## 2단계. 샘플 5개 먼저 변환

전체 자동화 전에 5개 정도만 먼저 테스트한다.

확인할 것:

- 한컴오피스에서 정상적으로 열리는지
- “상위 버전에서 작성한 파일입니다” 같은 팝업이 뜨는지
- 다른 이름으로 저장 시 HWPX 형식 선택이 가능한지
- 저장된 `.hwpx` 파일이 정상적으로 다시 열리는지

샘플에서 문제가 없으면 전체 변환으로 넘어간다.

## 3단계. 자동화 방식

자동화 흐름은 아래 순서로 고정한다.

```text
1. HWP 파일 열기
2. 팝업이 있으면 확인 처리
3. 다른 이름으로 저장
4. 파일 형식: HWPX 선택
5. 출력 폴더에 저장
6. 문서 닫기
7. 다음 파일 반복
```

핵심은 한컴오피스가 실제로 파일을 열고 저장해야 한다는 점이다.  
이 과정을 거치면 우리가 원하는 실제 HWPX 파일이 된다.

## 4단계. 자동화 시 주의할 점

자동화가 멈추는 주요 원인:

- “상위 버전에서 작성한 파일입니다” 팝업
- 보안 경고 팝업
- 읽기 전용 문서
- 손상된 HWP 파일
- 암호가 걸린 문서
- 같은 이름의 HWPX 파일이 이미 존재하는 경우
- 한컴오피스 프로세스가 백그라운드에서 멈추는 경우

처리 원칙:

- 실패한 파일은 건너뛰고 로그에 남긴다.
- 실패 파일만 나중에 수동으로 다시 확인한다.
- 자동화 중 멈추면 한컴 프로세스를 종료한 뒤 마지막 성공 파일 이후부터 재시작한다.
- 원본 HWP 파일은 절대 삭제하지 않는다.

## 5단계. 변환 결과 확인

전체 변환 후 아래를 확인한다.

```text
원본 HWP 개수
변환된 HWPX 개수
변환 실패 파일 수
HWPX 파일이 다시 열리는지 여부
```

로그에는 최소한 아래 항목을 남긴다.

```text
file_name
source_path
output_path
status
error_message
converted_at
```

상태값 예시:

```text
success
failed_open
failed_save
skipped_exists
needs_review
```

## 6단계. 전달 기준

다른 컴퓨터로 전달할 때는 아래만 공유한다.

```text
data/hwpx_690/
outputs/hwpx_conversion/conversion_log.csv
outputs/hwpx_conversion/conversion_failed.csv
```

공유 전 확인:

- `.hwpx` 파일 개수가 예상과 맞는지
- 실패 파일 목록이 따로 정리되어 있는지
- 원본 `.hwp` 파일이 실수로 섞이지 않았는지
- 파일명이 원본과 대응 가능한지

## 시간 추정

690개 기준 예상:

- 최소: 30분~1시간
- 중앙: 1~2시간
- 최대: 3~5시간

근거:

- 파일마다 열기/저장 시간이 다르다.
- 팝업이 많으면 자동화가 자주 멈출 수 있다.
- 큰 표나 이미지가 많은 문서는 저장 시간이 길어질 수 있다.
- 손상 파일이나 암호 파일은 수동 확인이 필요하다.

## 완료 기준

- 원본 HWP를 단순 rename하지 않았다.
- 한컴오피스에서 실제 HWPX 형식으로 저장했다.
- 변환 성공/실패 로그가 남아 있다.
- 실패 파일은 별도로 확인 가능한 상태다.
- 변환된 HWPX 파일이 한컴오피스에서 다시 열린다.
